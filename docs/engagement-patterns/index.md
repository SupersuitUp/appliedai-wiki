# Engagement Patterns

*Anonymized lessons from real implementation engagements. What worked, what did not, why. Pattern-first, not narrative-first.*

---

Engagement patterns are not case studies as marketing. They are field notes: specific, anonymized, and ruthlessly focused on what a future operator can do differently because someone else has already paid the cost of learning it.

Each entry follows the same shape: context → what we did → what worked → what did not work → the transferable lesson stated as an action.

## How the section is shaped

- Each pattern is sanitized: no client names, no industries that would identify the client, no engagement-specific numbers that betray identity.
- Lessons are stated as actions, not morals. "Insist on direct read access to the production output before scoping" beats "communication matters."
- Patterns are organized by the phase of the engagement they apply to (scoping, kickoff, week one, mid-engagement, handoff).
