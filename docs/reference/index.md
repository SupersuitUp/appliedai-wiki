# Reference

*Tools, standards, glossary, voice rules, quality rubric. The infrastructure pages that support the rest of the wiki.*

---

The Reference section holds the load-bearing utility pages: the tools you reach for, the standards that define what a deliverable looks like, the glossary that one-liners every coined term, and the rules that keep the writing and the bar consistent over time.

- **[Tools](/reference/tools)**: one page per tool. Follow the `What it is / What it costs / When to use it (and when not) / Setup pointer` template.
- **[Standards](/reference/standards)**: the canonical specs (ALIGN.md, INTEGRATE.md) that every engagement produces and every implementer follows.
- **[Voice Rules](/reference/voice-rules)**: the writing constraints every page is held to.
- **[Quality Rubric](/reference/quality-rubric)**: how we hold this wiki to a bar. Seven dimensions, mechanical checks, the gut test.
- **[Glossary](/reference/glossary)**: one-line-per-term summary across the whole wiki.
