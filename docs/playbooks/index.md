# Playbooks

*Opinionated, step-by-step procedures for the high-leverage moves of an implementation engagement. Each playbook produces a concrete outcome from a defined starting state.*

A playbook here is not a tutorial. It is a procedure with prerequisites, sequenced steps, an explicit success state, and named failure modes. You should be able to run a playbook end to end without external context and know whether it worked.

## How playbooks are shaped

- **When to run this.** The specific situation the playbook is designed for.
- **Prerequisites.** What must already be true.
- **Steps.** Numbered, atomic, observable.
- **What good looks like at the end.** The success state.
- **Common failure modes.** Failure → diagnostic → fix.

If your situation does not match the "when to run this," the playbook will not work and the right next move is elsewhere.
