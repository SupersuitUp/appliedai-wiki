# Vice Is Quantifiable

*Vice has a clear objective function. Virtue does not. This is why AI monetizes in vice first, and why any attempt to build a wisdom-optimizing AI runs into a fundamental mathematical floor.*

---

## The mechanism

Every AI model optimizes an objective function: a mathematical description of what good looks like. For some applications, the function is obvious and computable. Does the code run? Did the user click? Did they come back? Did they stay longer? These are measurable.

Vices are naturally measurable. Addiction is a feedback loop: the dopamine response, the return visit, the escalating engagement. The objective function for a pornographic AI is clear. The objective function for a casino-style engagement loop is clear. The objective function for an AI companion optimized to create emotional dependency is clear. All of these can be measured in real time with direct feedback from user behavior.

Virtue is not measurable the same way. What is the quantified signal that someone became wiser this week? That they made a harder, better choice than they would have six months ago? That their relevance-hierarchy shifted toward the good? There is no clean metric. And because there is no metric, there is no objective function, and there is no gradient to climb.

Sam (the host of the June 2026 discussion between Vervaeke and Pageau) framed it cleanly: "Vice is easier to quantify than virtue." Vervaeke extended it: "Reality is much harder to get to than illusion. That's why science is hard." Pageau added: "Virtue requires you to concentrate, to focus, to aim above. The vices just pull you. They're addictions."

## The pixelation problem

Vervaeke offered a second framing: even if you try to mathematically approximate virtue, the approximation pixelates it. The moment the good becomes a metric, you are no longer tracking the good. You are tracking the metric. And metrics, once targets, become gameable in ways the original good was not.

A "helpfulness score" for an AI companion is not helpfulness. A "user growth" metric for an educational tool is not education. The mathematical representation of the goal introduces gap between the map and the territory, and that gap is the space where vice lives. The features of the system that maximize the metric but not the goal are almost always vice-adjacent: they create the feeling of satisfaction without the substance.

## The monetization implication

AI platforms face the same pressure every attention-based platform faces. The business model rewards engagement. Engagement is measurable. Engagement maps more cleanly onto vice than virtue. The subscription model, the freemium model, the ad model: all reward keeping users on, keeping users coming back, keeping users emotionally activated.

Pageau argued this is structural, not incidental: "The very model that we're using to create these tools is leading us into our vices. So I don't see a solution out of that one."

The practical consequence for operators: the AI tools you select for your stack are not neutral. They were built inside incentive structures that rewarded vice-adjacent engagement patterns. The defaults, the prompts, the UX patterns, the model behaviors: all are downstream of an objective function optimized for retention, not for your actual goals.

## The operator response

The only exit is to impose your own objective function. That means:
- Defining what success looks like in your workflow with enough specificity to distinguish it from engagement
- Applying the [generate vs discriminate](/perspectives/llm-psychosis) discipline to every AI-produced output
- Using AI in contexts where you can measure actual quality, not just output volume

The wisdom-AI problem is unresolved at the AI-design level. At the operator level, the response is to build systems where the human supplies the objective function rather than inheriting the one the platform provides.

> **The platform's objective function is the one it was paid to optimize. Build yours explicitly, or inherit theirs by default.**
